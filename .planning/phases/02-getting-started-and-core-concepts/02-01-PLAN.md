**Page structure (prose sections + CodeGroup):**

1. **Opening paragraph (2-3 sentences):** Protochain is a gRPC API. Unlike REST where you construct URLs and call them directly, gRPC requires you to create a channel to the server and instantiate a typed client stub before making any calls. This page shows that setup for each supported SDK.

2. **"What you'll need" section:** A `<Note>` callout listing prerequisites: the appropriate language SDK installed, the Protochain server endpoint (host:port), and whether TLS is required for your environment.

3. **"Connection setup" section with CodeGroup (tab order: Go, Rust, TypeScript — per D-10):**

For **Go tab**, show the complete setup with comments explaining each step:
```go
package main

import (
    "log"

    "google.golang.org/grpc"
    "google.golang.org/grpc/credentials/insecure"
    account_v1 "github.com/meshtrade/protochain/lib/go/protochain/solana/account/v1"
)

func main() {
    // Create a channel to the Protochain server
    conn, err := grpc.NewClient("YOUR_ENDPOINT:50051",
        grpc.WithTransportCredentials(insecure.NewCredentials()),
    )
    if err != nil {
        log.Fatalf("failed to connect: %v", err)
    }
    defer conn.Close()

    // Instantiate a client stub for the service you want to call
    // Replace account_v1 with the service package you need
    client := account_v1.NewServiceClient(conn)
    _ = client // use client to make calls
}
```

For **Rust tab**:
```rust
use protochain::solana::account::v1::service_client::ServiceClient;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    // Connect to the Protochain server
    // The URL scheme (http:// vs https://) controls TLS
    let mut client = ServiceClient::connect("http://YOUR_ENDPOINT:50051").await?;

    // client is ready to use
    Ok(())
}
```

For **TypeScript tab**:
```typescript
import * as grpc from '@grpc/grpc-js';
import { ServiceClient } from '@protochain/solana-account-v1';

// Create a client — credentials.createInsecure() for plaintext connections
// Use credentials.createSsl() for TLS-secured connections
const client = new ServiceClient(
  'YOUR_ENDPOINT:50051',
  grpc.credentials.createInsecure()
);
```

4. **"Switching services" section (short):** Each Protochain service has its own generated client stub. To use a different service, import that service's package and instantiate its client using the same `conn` (Go) or by calling `.connect()` again (Rust/TypeScript). Example import paths for each language:
  - Go: `github.com/meshtrade/protochain/lib/go/protochain/solana/[service]/v1`
  - Rust: `protochain::solana::[service]::v1::service_client::ServiceClient`
  - TypeScript: `@protochain/solana-[service]-v1`

5. **"TLS and credentials" section (short):** The examples above use plaintext connections (no TLS). For production environments, replace `insecure.NewCredentials()` (Go) or `http://` (Rust) or `credentials.createInsecure()` (TypeScript) with your TLS credentials. Contact the Protochain team for endpoint and TLS configuration for your environment.
