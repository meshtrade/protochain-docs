**Opening paragraph:** One short paragraph — creating a token on Solana requires three steps: create a mint account, create a holding account for the recipient, and mint tokens to it. This guide covers both SPL Token and Token-2022 paths.

**Section: "Choose your token program"**
This section must appear BEFORE any code (per D-04 — make the SPL vs Token-2022 choice obvious and early).

Two-sentence lead: Both token programs use the same Protochain API shape — you choose the program once, at mint creation, and it cannot be changed. See [Token Programs](/concepts/token-programs) for the full trade-off comparison.

Present the choice as a small comparison table:

| | SPL Token | Token-2022 |
|---|---|---|
| Metadata | Metaplex PDA (external) | On-chain extension (native) |
| Extensions | Not supported | Transfer fees, interest-bearing, etc. |
| Compatibility | Maximum ecosystem support | Newer wallets/DEXs only |

Then note: **If you need extensions or native on-chain metadata, use Token-2022. If you need maximum wallet compatibility, use SPL Token.**

**Section: "Prerequisites"**
- A funded payer account
- A generated mint keypair (the mint's public key). See [Quickstart](/guides/quickstart).
- A Protochain connection (see [Connecting to Protochain](/guides/connecting)).

**Section: "Create the mint"**
Prose: Call [CreateToken2022Mint](/api-reference/token-program/mint-creation) or [CreateSPLTokenMint](/api-reference/token-program/mint-creation) depending on your token program choice. Both return a list of `instructions` — include all of them in the transaction (the service handles ordering).

Use a two-group structure here: show Token-2022 first, then SPL Token. Each gets its own H3 subheading and CodeGroup (Go, Rust, TypeScript).

**H3: Token-2022 mint**
CodeGroup showing CreateToken2022Mint call:
- `payer_pub_key`: payer's public key
- `mint_pub_key`: the new mint account's public key (you generated this keypair)
- `mint_authority_pub_key`: who can mint tokens (use payer for simplicity)
- `decimals`: 6 (comment: "like USDC; use 0 for NFTs")
- `extensions`: empty list for basic mint (comment: "add Token2022Extension values here for transfer fees, etc.")
Show the response `instructions` being collected.

**H3: SPL Token mint**
CodeGroup showing CreateSPLTokenMint call with same fields. Show `metadata` field as optional — include a commented-out example with name/symbol/uri for Metaplex metadata.

**Section: "Create a holding account"**
Prose: Before minting, the recipient needs a token holding account (Associated Token Account) for this mint. Call [CreateToken2022HoldingAccount](/api-reference/token-program/holding-accounts) or [CreateSPLTokenHoldingAccount](/api-reference/token-program/holding-accounts) — use the same token program as the mint.

Add a Note callout: Pass `owner_pub_key` (the recipient's system wallet address), NOT a pre-computed ATA address. Protochain derives the ATA automatically.

Two H3 subheadings (Token-2022 / SPL Token), each with a CodeGroup. Token-2022 shows empty `extensions` list (comment: "add holding account extensions here"). SPL Token shows the minimal 3-field call.

**Section: "Compile and submit the mint transaction"**
Prose: Collect all instructions from the mint creation response, the holding account response, and assemble them into a single transaction. Compile with [CompileTransaction](/api-reference/transaction/compile-transaction), sign with both the fee payer key and the mint keypair (both are signers), then submit.

Add a Note callout: The mint account keypair MUST be included as a signer in SignTransaction — it is not just a public key reference.

CodeGroup (Go, Rust, TypeScript) showing:
1. Assembling all instructions into one Transaction
2. CompileTransaction call
3. SignTransaction with both private keys
4. SubmitTransaction, capturing signature

Keep snippets focused — one language's code should be ~15-20 lines max.

**Section: "Mint tokens"**
Prose: With the mint account created, call [Mint](/api-reference/token-program/mint) to issue tokens. Mint is token-program agnostic — Protochain reads the mint on-chain to determine the token program automatically.

Add a Note callout: Pass `destination_owner_pub_key` as the recipient's system wallet address (NOT the ATA address). The ATA is derived automatically.

CodeGroup (Go, Rust, TypeScript) showing the Mint call:
- `mint_pub_key`: the mint address
- `destination_owner_pub_key`: recipient's wallet address
- `amount`: `"1000.0"` (comment: "human-readable; 1000 tokens at 6 decimals = 1,000,000,000 base units")

Then compile, sign (mint authority only this time), and submit — can reference back to the compile/sign/submit pattern from the previous section (brief, no full repeat).

- Full Token Program reference: [Token Program Service](/api-reference/token-program/overview)
- Monitor your mint transactions: [Monitor Transactions guide](/guides/monitor-transaction)
